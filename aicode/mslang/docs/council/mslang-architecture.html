系统架构概览

Maple 采用经典的编译器-虚拟机分离架构，源码经过词法分析、单遍编译生成字节码， 再由栈式虚拟机解释执行

点击上方模块查看详情

Maple 的执行流程如上图所示：源码首先经过 Scanner 进行词法分析生成 Token 流， 然后 Compiler 使用 Pratt Parser 进行单遍编译直接生成字节码， 最后 VM 通过指令分发循环解释执行字节码。GC 模块负责管理所有堆分配对象的生命周期。

执行流程详解

Scanner 工作原理

Scanner 是一个按需生产的词法分析器（Lazy Lexer），它不会一次性扫描整个源码， 而是在 Compiler 需要时逐个返回 Token。这种设计配合单遍编译，内存效率高。

1 跳过空白字符

skip_whitespace() 跳过空格、换行、注释

2 识别 Token 类型

根据首字符判断：标识符、数字、字符串、符号

3 关键字检测

identifier_type() 使用 Trie 结构匹配关键字

Token 类型示例
TOKEN_PRINT → "print"
TOKEN_STRING → "hello"
TOKEN_SEMICOLON → ;
TOKEN_EOF → 文件结束

Pratt Parser 单遍编译

Compiler 采用 Pratt Parser（也称为 Top-Down Operator Precedence）， 将表达式解析与代码生成合二为一，边解析边生成字节码，无需构建 AST。

栈式虚拟机执行

VM 采用栈式架构，所有操作都在值栈上进行。指令短小紧凑， 无需显式操作数地址，操作数隐式位于栈顶。

VM 指令分发循环

VM::run() - 核心执行循环
hot path 
InterpretResult VM::run() {
  CallFrame* frame = &frames_[frame_count_ - 1];

  // 无限循环，直到遇到 OP_RETURN 或错误
  for (;;) {
    u8_t instruction = READ_BYTE();  // *frame->ip++
    
    switch (static_cast<OpCode>(instruction)) {
      case OpCode::OP_CONSTANT: {
        Value constant = READ_CONSTANT();
        push(constant);
        break;
      }
      
      case OpCode::OP_ADD: {
        if (peek(0).is_string() && 
            peek(1).is_string()) {
          concatenate();  // 字符串拼接
        } else {
          double b = pop().as_number();
          double a = pop().as_number();
          push(Value(a + b));
        }
        break;
      }
      
      case OpCode::OP_CALL: {
        int arg_count = READ_BYTE();
        if (!call_value(peek(arg_count), arg_count)) {
          return INTERPRET_RUNTIME_ERROR;
        }
        frame = &frames_[frame_count_ - 1];
        break;
      }
      
      case OpCode::OP_RETURN: {
        Value result = pop();
        close_upvalues(frame->slots);
        frame_count_--;
        if (frame_count_ == 0) {
          pop();  // 弹出脚本函数
          return INTERPRET_OK;
        }
        push(result);
        frame = &frames_[frame_count_ - 1];
        break;
      }
      // ... 40+ 其他指令
    }
  }
}

READ_BYTE()

宏定义: (*frame->ip++) 读取当前指令并递增指令指针

调用帧切换

OP_CALL 时 frame_count_++，OP_RETURN 时 frame_count_--， frame 指针始终指向当前帧

Dispatch 方式

使用 C++ switch-case 实现 Direct Threading， 现代编译器会优化为跳转表，效率高

辅助宏

READ_SHORT() → 2字节
READ_CONSTANT() → 常量池
BINARY_OP(op) → 二元运算
