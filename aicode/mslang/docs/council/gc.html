GC 垃圾回收器

采用 三色标记-清扫 (Tri-color Mark-and-Sweep) 算法， 追踪式垃圾回收。暂停式执行，简单但正确，适合教学和小型语言实现。

GC 算法详解

1

标记阶段

从所有 GC Roots 开始，将直接引用的对象标记为 Gray，加入工作列表。

2

追踪阶段

从 Gray Stack 中取出对象，标记其引用的子对象，直到没有 Gray 对象。

3

清扫阶段

遍历对象链表，回收未标记的 White 对象，重置 Black 对象的标记位。

}

GC Roots 来源

S
值栈 (VM Stack)
所有在栈上的 Value，包括临时计算结果
G
全局变量 (Globals)
globals_ 哈希表中所有的全局变量
F
调用帧 (Frames)
所有激活的 CallFrame 中的闭包
U
打开的上值 (Open Upvalues)
指向栈上变量的 Upvalue 链表
C
编译器 (Compiler)
当前正在编译的函数对象

GC 触发与调优

触发条件

// 每次分配时检查
template <typename T>
T* allocate(...) {
bytes_allocated_ += sizeof(T);
if (bytes_allocated_ > next_gc_) {
collect_garbage();
}
return new T(...);
}
GC 后调整阈值
next_gc_ = bytes_allocated_ * kGC_HEAP_GROW;
// kGC_HEAP_GROW = 2 (默认)

字符串表清理

Sweep 阶段之前需要清理字符串驻留表中指向已回收字符串的条目， 防止 dangling pointer。

// 清理字符串表
void Table::remove_white() {
for (auto& entry : entries_) {
if (entry.key && !entry.key->is_marked()) {
remove(entry.key);
}
}
}
